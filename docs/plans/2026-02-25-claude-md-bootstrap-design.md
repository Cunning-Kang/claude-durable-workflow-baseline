# CLAUDE.md 自动化流程设计（Skill 方案）

## 1. 背景与目标

当前流程依赖手动复制模板并回填项目命令，重复操作多且容易遗漏。目标是提供一个可全局复用的 Skill，在任意项目中一键完成 `CLAUDE.md` 的生成与更新。

本设计以“模板内置”作为核心决策：模板内容直接放入 Skill，不在运行时读取 iCloud 文件。

## 2. 设计决策

### 2.1 采用方案

采用方案 A：**内置模板到 Skill**。

- 运行时使用 Skill 内置模板渲染 `CLAUDE.md`
- 执行 `/init`（或等价初始化步骤）获取项目上下文
- 从初始化结果自动提取命令并回填 `Project Overrides`
- 自动覆盖项目根 `CLAUDE.md`

### 2.2 不采用方案

不采用“每次运行都读取 iCloud 模板”。原因：

- 依赖个人路径，迁移性差
- 受外部同步状态影响，稳定性差
- 增加运行时失败面，不利于自动化

## 3. 目标范围与非目标

### 3.1 目标范围

- 在项目根生成或覆盖 `CLAUDE.md`
- 固定写入模板结构与默认策略
- 自动回填命令字段：
  - `ENV_SETUP_CMD`
  - `TEST_CMD`
  - `LINT_CMD`
  - `TYPECHECK_CMD`
  - `BUILD_CMD`
- 保留顶部导入行：`@~/.claude/standards/universal-claude.md`

### 3.2 非目标

- 不自动提交 git
- 不修改业务代码文件
- 不从外部 URL 或 iCloud 读取运行时模板

## 4. 架构设计

### 4.1 组件

1. **Skill 主流程（claude-md-bootstrap）**
   - 驱动完整流水线：初始化、提取、渲染、写入

2. **命令提取器**
   - 从 `/init` 结果中抽取命令字段
   - 仅提取可确认值，不做猜测

3. **模板渲染器**
   - 用提取结果回填 `Project Overrides`
   - 维护模板固定结构

4. **写入与回滚模块**
   - 写入前备份 `CLAUDE.md.bak`
   - 写入失败自动恢复

## 5. 数据流

1. 检测项目根与目标文件位置
2. 读取并备份已有 `CLAUDE.md`（若存在）
3. 执行 `/init` 收集项目初始化信息
4. 提取命令字段并生成映射
5. 渲染最终模板文本
6. 覆盖写入项目根 `CLAUDE.md`
7. 输出结果摘要（写入成功、已填字段、留空字段）

## 6. 字段映射规则

- 只回填可确定来源的命令
- 缺失字段保持空值
- 不生成或臆造命令
- 覆盖时保留模板中固定键名与顺序

## 7. 错误处理与回滚

### 7.1 硬失败（终止并回滚）

- 无法定位项目根
- 模板渲染失败（占位符异常）
- 文件写入失败

### 7.2 软失败（写入成功但提示）

- `/init` 成功但命令提取不完整
- 部分命令字段留空

### 7.3 回滚策略

- 写入前创建 `CLAUDE.md.bak`
- 写入失败自动恢复备份
- 写入成功后保留备份，便于人工 `diff`

## 8. 验收标准

以下条件需同时满足：

1. Skill 执行后，项目根 `CLAUDE.md` 存在且可读
2. 文件顶部包含 `@~/.claude/standards/universal-claude.md`
3. `Project Overrides` 包含完整键结构
4. 可提取命令已回填，不可提取命令保持空值
5. 失败场景可恢复到写入前状态

## 9. 运行输出规范

- **成功**：已写入 + 回填字段列表 + 留空字段列表
- **软失败**：已写入 + 待人工补全字段
- **硬失败**：未改动（已回滚）+ 明确失败原因

## 10. 后续实施建议

1. 先实现最小可用版本：模板内置 + 命令提取 + 自动写入 + 备份回滚
2. 再补充可选增强：提取日志更详细、字段诊断更清晰
3. 如需与模板源同步，新增“手动同步模板”维护命令，而不是运行时强依赖外部文件
