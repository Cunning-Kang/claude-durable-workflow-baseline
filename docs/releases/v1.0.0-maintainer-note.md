## C. Change-Routing Reminder

| What you're changing | Route to |
|---------------------|----------|
| Host-wide rule or constraint | `global/standards/core-standard.md` or `global/guides/` |
| Deterministic gate or enforcement | Hooks in `distribution/hooks/` |
| Protocol behavior definition | `baseline/docs/workflow/` |
| Reusable pattern or convention | `baseline/memory/patterns.md` |
| Recurring pitfall or gotcha | `baseline/memory/gotchas.md` |
| One-off note or context | Not persisted to baseline; stays in session memory |

**Rule of thumb:** If it needs to survive a session and affect future sessions, it goes in the right baseline surface — not `global/CLAUDE.md` and not session memory.

## E. Anti-Pattern Reminders

Continue avoiding these patterns established during the refactor:

1. **Don't let global core grow thick again.** `global/standards/core-standard.md` holds stable constraints only — not every useful rule.
2. **Don't let hooks become a second control plane.** Hooks are opt-in templates, not a runtime layer that orchestrates behavior.
3. **Don't stuff protocol content into memory.** Protocol definitions belong in `baseline/docs/workflow/`; memory holds patterns and gotchas.
4. **Don't write one-off notes into memory.** Session-specific context stays in session memory, not the baseline skeleton.
5. **Don't let refactor docs drift from actual repo surface.** When structure changes, update the handoff or initiate a new refactor closeout.
6. **Don't repeat the same fact across files.** One authoritative location per fact; cross-references over duplication.
7. **Don't write vague entries.** Memory and pattern entries without sufficient context or actionable guidance create noise without value. Each entry must earn its place.
