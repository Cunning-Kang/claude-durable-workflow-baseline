# 审查关卡语义 — PASS / FAIL / BLOCKED

> 这是 `core-standard.md` 中定义的审查关卡的规范语义。

本文件仅定义审查关卡状态。是否需要审查由 `core-standard.md` 决定。

---

## 独立性要求

独立意味着：审查者未实施所审查的变更。
除非更高优先级策略明确允许，否则自我审查不满足此关卡。
非独立：同一智能体的第二轮审查，或修改了实现代码的审查者。

## 必需证据

- `Reviewer`：<身份>
- `Reviewed scope`：<commit、diff 范围或确切文件>
- `Reference`：<包含审查结果的消息、任务、文件、PR 评论或其他产物>

审查证据仅在 `Reviewed scope` 与声称完成的当前变更匹配时有效。

## 状态定义

| 状态 | 条件 |
|------|------|
| `PASS` | 独立审查完成；记录证据中无阻塞性发现 |
| `FAIL` | 独立审查完成；记录证据中有阻塞性发现 |
| `BLOCKED` | 无法在当前条件下完成必需的独立审查并提供记录证据 |

缺乏独立审查和记录证据：`PASS` 和 `FAIL` 无效，`BLOCKED` 为必需。
当审查为必需时，`N/A` 无效。

## 阻塞状态规则

`BLOCKED` → 状态保持 `In Progress`，除非更高优先级策略明确允许替代方案。
进入 `BLOCKED` 时，说明解除阻塞所需条件。
